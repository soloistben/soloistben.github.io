spring学习之注解篇（二）

Spring学习 之 Spring AOP 注解

＊AspectJ 注解

  • @AspectJ 的风格类似纯 Java 注册的普通 Java类
  • Spring 可以使用 AspectJ 来做切入点注解
  • AOP的运行时仍旧时纯的Spring AOP，对AspectJ 的编译器或者织入五依赖性
  • 对@Aspect 可支持 XML或者 Java 风格的配置
  • 拥有@Aspect的任何bean将被Spring自动识别并应用
  • 用@Aspect注解的类可以有方法字段，他们也可以包括 Pointcut 切入点、Advice 通知、Introduction引入 的声明
  • 一个类中的@Aspect注释表示它是一个切面，并且将自己从自动代理中排除

  • @Aspect注解时不能通过类路径自动检测发现的，所以需要配合使用@Component注解或者在xml配置bean

  • @Pointcut注解切入点——切入点表达式 通过 &&、||和 ！进行组合，也可以通过名字引用切入点表达式（通过组合可以建立更加复杂的切入点表达式（但不建议写太复杂，应该清晰明朗））

  • 一个切入点通过一个普通方法定义来提供，@Pointcut注解，方法返回类型必须为void

  • 定义良好的 pointcut

    1. AspectJ是编译期的AOP
    2. 检查代码并匹配连接点于切入点的代价是昂贵的
    3. 一个好的切入点应该包括以下几点：
      • 选择特定类型的连接点，如：execution,get,set,call,handler
      • 确定连接点范围，如：within,withincode
      • 匹配上下文信息，如：this，target，@annotation

  • @Before

  • @AfterReturning——指明切入点pointcut=”pointcut()”和返回值returning=”returnValue”

  • @AfterThrowing
  • @Around——第一个参数必须是ProceedingJoinPoint类型

  • @After

  • Advice扩展

    1. 携带参数
    2. 携带注释

    • 自定义注解

    • 用自定义的注解去注解方法

Spring学习 之 Bean注解

＊Bean管理的注解

  • 类的自动检测及Bean注册——通过自动检测类并注册Bean到xml配置文件中

  • xml配置—-<context:annotation-config/>

    • xml使用context的基本设置
      1. 在beans引入xmlns：context地址，引入context的xsi:schemaLocation地址（可在左下角Namespaces勾选）
      2. <context:component-scan base-package=””> </context:component-scan/> 填入相对应包的名称（扫描整个包下面的所有注解，也可以用过滤器过滤不需要扫描的文件）
      3. 可直接配置代理方式，有三个可选值：no，interfaces，targetClass（默认是no）

  • Bean常用注解 （使用 java 定义 bean）

    • @Component——一个通用的注解，可用于任何bean（@Repository、@Service、@Service是@Component的子注解）
    • @Repository——通常用于注解DAO类，即持久层（与数据库连接）
    • @Service——通常用于注解Service类，即服务层
    • @Controller——通常用于Controller类，即控制层（MVC）
    • @Required——适用于bean属性的setter方法（注解在setter上）（不常用）
    • @Autowired——适用于成员变量、setter方法、构造器上（常用）（@Autowired(required=false)可避免找不到bean而抛出异常，但在注解构造器时，每个类只能有一个构造器（一个类可有多个重载的构造器）被标记required=true。@Autowired是由Spring BeanPostProcessor处理的，所以不能在自己的BeanPostProcessor或者BeanFactoryPostProcessor类型应用这些注解，这些类型必须通过 xml 或者Spring的@Bean注解加载）
    • @Qualifier——-可在@Autowired下用于缩小赋值范围（可用于成员变量、函数参数）（在不用注解的情况下，可在 xml 对应 bean 下进行配置也可达到同样效果）
    • @Resource——适用于成员变量、只有一个参数的setter方法，所以在目标是构造器或者一个多参数方法时，最好的方式时使用@Qualifier

  • 可通过元注解自定义注解

spring学习之基础篇（二）

Spring学习 之 Bean与IOC的基础

＊Spring 注入方式

  • Spring注入是指在加载 bean 配置同时，完成对变量的赋值
  • 注入之后，则可在service中的业务操作执行dao的数据操作。
    常用的两个注入方法：

  • 构造注入 (Construct构造函数)

    1. 首先创建一个 InjectionServicre 接口和一个需要实例化的 InjectionDAO 接口，创建一个应用类 InjectionServiceImpl 实现接口，造构造方法中给一个在 xml 以配置的 bean 实例赋值。

    2. xml中的配置

  • 设置注入 (setter方法)

    1. 在构造注入条件下，只需要一个 setter 方法

    2. 修改xml配置

＊Bean 作用域

  • singleton:单例，指一个Bean容器中只存在一份（即使多次getBean同一个bean的id，都表示是同一个bean实例，可用hashcode验证）
  • prototype：每次请求（每次使用）创建新的bean实例，bean的destroy方式不生效（也就是每次getBean同一个bean的id，都是不相同的bean实例）
  • request：每次http请求创建一个实例且仅当前request内有效
  • session：每次http请求创建，当前session内有效
  • global session：基于portlet的web中有效（portlet定义了global session），如果是在web中，同session一样功能

  1. 创建一个类

  2. xml配置

＊Bean 生命周期

  • Bean的定义 -> 初始化 -> 使用 -> 销毁

    1. Bean的定义是在 xml 中配置所创建的类

    2. 初始化会在注入时开始，销毁会在使用后随着context销毁而随之销毁。

      • 初始化和销毁有三种方式
        1. 直接在bean中配置——init-method=”start” destroy-method=”stop”—— start 和 stop是方法的名称
        2. 实现接口配置——implements InitializingBean,DisposableBean—— 实现两个接口，重写 afterPropertiesSet 和 destroy 方法
        3. 全局变量配置——在 xml 的 beans 中配置 default-init-method=”defaultInit” default-destroy-method=”defaultDestory”—— defaultInit 和 defaultDestory 是方法的名称

＊Aware接口

  • 以 Aware 为结尾的接口，实现了Aware接口的类在 bean 初始化后，可获得相应的资源。
  • 通过 Aware 接口，可以对 Spring 相应资源进行操作（一定要慎重，尽量避免使用）
  • 为对 Spring 进行简单的扩展提供了方便的入口
Aware 为结尾的接口的两个例子：

  1. ApplicationContextAware

    • 在知道在 xml 配置某个 bean 的 id 后，可通 id 实例 bean 的对象。

    • 在别处获得该 bean 实例，是同一个 bean 实例，可通过 hasCode 来验证。

  2. BeanNameAware

    • 可在实现 BeanNameAware 接口的类获得其在 xml 配置的 bean 的 id。

    • 同时实现ApplicationContextAware BeanNameAware两个接口，可在不需要知道在xml配置的bean的id，可在该类本身获取bean实例。

＊Bean的自动装配——Autowiring

  • Autowiring 是一个 Beans 的全局属性

  • Autowiring 的四种结果：

    1. No ：不做任何操作
    2. byName：根据属性名自动装配。根据名字查找与属性完全一致的 bean，并将其与属性自动装配
    3. byType：如果容器中存在一个与指定属性类型相同的 bean ，那么将于该属性自动装配；如果存在多个该类型 bean，则抛出异常，并指出不能使用byType方式进行装配；如果找不到相匹配的bean，则什么事都不发生。

    4. Constructor：与 byType 方式类似，不同之处在于他应用于构造器参数。如果容器中没有找到与构造器参数类型一致的 bean，那么抛出异常。

＊Resources

  • 针对于资源文件的统一接口

  • 支持多种路径形式：

    1. UrlResource:URL对应的资，根据一个URL地址即可构建（url:http://xxx）

    2. ClassPathResource：获取类路径下的资源文件（classpath:xxx.db）

    3. FileSystemResource：获取文件系统里面的资源（file:C:\\xxx\\xxx.db）

    4. ServletContextResource：ServletContext封装的资源，用于访ServletContext环境下的内容

    5. InputStreamResource：针对于输入流封装的资源

    6. ByteArrayResource：针对于字节数组封装的资源

spring学习之基础篇（一）

Spring学习 之 Bean、IOC、AOP

Spring 学习需掌握的基础:

  • IOC (配置与注解)
  • Bean (配置与注解)
  • AOP (配置 注解 AspectJ 与 API)

＊Spring 是什么?

  1. Spring 是一个开源框架,为了解决企业应用开发的复杂性而创建的.
  2. 轻量 —— (对于大小与开销都是轻量级的)
  3. 轻量级的 IOC 控制反转 —— (降低类之间的耦合,面向接口编程)
  4. 面向切面的 AOP —— (分离应用的业务逻辑 与 系统级服务进行内聚性 的开发)
  5. 容器 —— (包含并管理应用对象的配置和生命周期)

＊IOC

  • 控制反转（IOC）与依赖注入（DI）是同一个概念
    • 接口注入 (interface接口)
    • 构造注入 (Construct构造函数)
    • 设置注入 (setter方法)
  • IOC：控制反转 控制权的转移，应用程序本身不负责以来对象的创建和维护，而是由外部容器负责创建和维护
  • 一个对象依赖的其它对象会通过被动的方式传递进来，而不是这个对象自己创建或者查找依赖对象.
  • 通过 面向接口编程 来降低类之间的耦合,提高系统可插入、可测试、可修改等特性.
  • 面向接口编程 可分清层次结构,每层只向上层提供接口,各层次之间仅仅以来接口,不是实现类.当下层需要改变时，只要接口及接口功能不变，则上层不用做任何修改.

＊Bean

  • IOC 容器可以看成是 Beans 关系的一个集合,Context 作用就是为 Bean 提供一个运行时环境.
  • Bean 的定义(由使用者完成)
  • Bean 的创建 和 Bean 的解析(Spring完成)

＊你的第一个 bean

  1. 打开IDE -> 创建 java 项目 -> 导入spring基本的jar包(spring学习之工具篇有说明)

  2. 新建一个 Helloworld 类

  3. 新建xml配置文件

  4. 新建一个测试类

＊AOP

  • Aspect Oriented Programming 的缩写，表示面向切面编程，通过预编译方式（AspectJ）和运行期动态代理（方式有 SpringAOP、JbossAOP，代理有 JDK动态代理、CGLib动态代理）实现程序功能的统一维护的一种技术

  • 主要功能是：日志记录，性能统计，安全控制，事务处理，异常处理等

  • 须了解几个相关概念：

名称 英文名称 说明
切面 Aspect 一个关注点的模块化，这个关注点可能会横切多个对象
连接点 Joinpoint 程序执行过程中的某个特定点
通知 Advice 在切面的某个特定的连接点上执行的动作
切入点 Pointcut 匹配连接点的断言，在AOP中通知和一个切入点表达式关联
引入 Introduction 在不修改类代码的前提下，为类添加新的方法和属性
目标对象 Target Object 被一个或者多个切面所通知的对象
AOP代理 AOP Proxy AOP框架创建的对象，用来实现切面契约（Aspect Contract）（包括通知方法执行等功能）
织入 Weaving 把切面连接到其他的应用程序类型或者对象上，并创建一个被通知的对象，分为：编译时织入、类加载时织入、执行时织入
  • Advice通知类型
名称 英文名称 说明
前置通知 Before advice 在某连接点(Join Point)之前执行的通知，但不能阻止连接点前的执行（除非它抛出一个异常）
返回后通知 After Returning advice 在某连接点(Join Point)正常完成后执行的通知
抛出异常后通知 After throwing advice 在方法抛出异常退出时执行的通知
后通知 After advice 在某连接点(Join Point)退出的时候执行，无论是否抛出异常
环绕通知 Around advice 包围一个连接点(Join Point)的通知

  • SpringAOP：纯Java实现，无需特殊的编译过程，不需要控制类加载层次；目前只支持方法执行连接点（通知Spring Bean的方法执行）；侧重于提供一种AOP实现与Spring IOC容器之间的整合，用于帮助解决企业应用中常见的问题

  • SpringAOP 不是一个完整的AOP，只立足于Spring

  • AspectJ 是综合全面的AOP

  • 有接口：SpringAOP 默认使用标准的的 Java SE 动态代理作为AOP代理，这使得任何接口都可以被代理

  • 无接口：SpringAOP 中也可以使用CGLIB代理（如果一个业务对象并没有实现一个接口）

    2. 在窗口的 work with 中输入 http://download.eclipse.org/releases/neon (neon是Eclipse版本名) 注：下载路径一定要与版本号相对应，否则会安装过慢甚至安装失败。
    3. 勾选以下4个选项，然后 next 到安装完重启。
      • Eclipse Java EE Developer Tools
      • Eclipse Java Web Developer Tools
      • Eclipse Web Developer Tools
      • Eclipse XMl Editors and Tools

  • IDEA UItimate

    IntelliJ IDEA 是java语言开发的集成环境，IntelliJ在业界被公认为最好的java开发工具之一，尤其在智能代码助手、代码自动提示、重构、J2EE支持、各类版本工具(git、svn、github等)、JUnit、CVS整合、代码分析、 创新的GUI设计等方面的功能可以说是超常的。
    IntelliJ IDEA UItimate 是付费版，集合了各种强大插件，而社区免费版没有，但可以尝试。

2、Spring-tool-suite

  • Spring-tool-suite 在 MyEclipse 打开
    • 右击项目 -> MyEclipse -> Project Facets -> install spring facet
  • Spring-tool-suite 主要安装与Eclipse
    • Windows：
      1. 打开 Eclipse -> Help -> Install New SoftWare
      2. Add -> Archive -> 选中下载好的 Spring-tool-suite 插件zip (必须对应当前使用的Eclipse版本)
      3. 导入安装

    • Linux:

      1. Spring-tool-suite 需要下载Linux对应的 tar.gz

      2. 打开终端

      3. 即可在 Eclipse 中打开 Spring IDE

3、Spring for framework

  • Spring for framework 在那个IDE用处都一致
    1. 下载所需要的 Spring for framework 版本
    2. 将解压目录下 libs 的 jar 复制项目新建的 lib 文件夹中 (至少要有4个基本jar包)
      • spring-beans-4.3.1.RELEASE.jar
      • spring-context-4.3.1.RELEASE.jar
      • spring-core-4.3.1.RELEASE.jar
      • spring-expression-4.3.1.RELEASE.jar
    3. 右击项目 -> Build Path ->Configure Build Path -> 将 lib 中的 jar 包全部加入 -> apply and close
